## 🎯 Objectif

Former des ingénieurs DevOps capables de :

- Programmer (Python) et scripter (Bash) pour l'automatisation
- Administrer un système Linux en production
- Gérer le code source avec Git
- Construire et orchestrer des conteneurs (Docker, Kubernetes)
- Provisionner et configurer l'infrastructure (Terraform, Ansible)
- Mettre en place des chaînes CI/CD
- Monitorer applications et infrastructure

## 🗂️ Structure d'un module

Chaque module suit le même format pédagogique :

```
NN-Nom-Module/Technologie/
├── Lecon-XXX.md              # cours théorique avec exemples
├── TP/
│   ├── TP1-Debutant.md       # ⭐    TP guidé étape par étape
│   ├── TP2-Intermediaire.md  # ⭐⭐   TP semi-guidé
│   └── TP3-Avance.md         # ⭐⭐⭐  TP en autonomie
└── Corriges/
    ├── Corrige-TP1.md        # code complet + explications
    ├── Corrige-TP2.md
    └── Corrige-TP3.md
```

## 🎓 Public visé

- Étudiants en **informatique** (Bac+2 à Bac+5)
- Professionnels en **reconversion** vers le DevOps
- Développeurs souhaitant **monter en compétences** en infrastructure
- Autodidactes suivant la **roadmap.sh** de leur côté

**Prérequis minimum :**
- Bases en informatique
- Un PC sous Windows, macOS ou Linux
- 10 Go d'espace disque pour les VMs / conteneurs

## 🚀 Pour démarrer

1. **Clonez** ce dépôt :
   ```bash
   git clone https://github.com/cloudarmelpro/devops-mastery.git
   cd devops-mastery
   ```

2. **Suivez la progression** dans l'ordre des modules numérotés.

3. **Pour chaque module** : lisez la leçon, faites le TP, comparez avec le corrigé.

4. **Validez vos acquis** : chaque TP a une grille d'évaluation chiffrée.
